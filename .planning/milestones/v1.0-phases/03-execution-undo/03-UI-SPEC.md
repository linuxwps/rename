### Accent Reserved For

Accent (#007AFF) is reserved for these exact elements only:
1. Execute button background (enabled state)
2. Spinner animation stroke/color during execution
3. Undo button text accent on hover
4. Toast close button (if close button is added)

Do NOT use accent color for: disabled execute button, undo button default state, conflict count text, result indicators (✓/✗), toast background, toast borders (except optional error accent).

Fixed-height bottom bar appended to RenamePanel:

```
┌──────────────────────────────────────────┐
│  .rename-panel (flex column, h: 100%)    │
│                                          │
│  ┌────────────────────────────────────┐  │
```

**Structure:**
- Container: `.action-bar` — full width, 48px height, `#fafafa` background, `1px solid #eee` top border
- Inner layout: `display: flex`, `align-items: center`, `justify-content: space-between`, padding `0 16px`
- Left slot: conflict count indicator (`.action-bar__conflicts`) — hidden when `totalConflicts === 0`
- Right slot: action buttons container (`.action-bar__actions`) — `display: flex`, `gap: 8px`

### Action Bar States

#### Before Execution (no conflicts)
```
│                                  [执行重命名]  │
```
Execute button: `#007AFF` bg, white text, `border-radius: 6px`, `padding: 8px 24px`, `font-size: 14px`, `font-weight: 600`, `border: none`, `cursor: pointer`.

#### Before Execution (with conflicts)
```
│  ⚠ 发现 N 个冲突               [执行重命名]  │
```
Execute button: `#e0e0e0` bg, `#999` text, `cursor: not-allowed`.
Conflict count: `#d32f2f` text, `font-size: 13px` (Label), `display: flex`, `align-items: center`, `gap: 4px`.

#### During Execution
```
│                                  [重命名中... ◌]  │
```
Execute button: same as disabled (or `opacity: 0.7`), text = "重命名中...", inline spinner (CSS-only), `cursor: wait`, `pointer-events: none`.

Spinner: CSS-only spinning circle, 16px × 16px, `border: 2px solid rgba(255,255,255,0.3)`, `border-top-color: #fff`, `border-radius: 50%`, `animation: spin 0.6s linear infinite`.

#### After Execution (success)
```
│  ✓ 重命名完成                   [撤销重命名]  │
```
Left slot switches to success status: `#155724` text, "✓ 重命名完成" (replaces conflict count).
Undo button: `#666` text, `1px solid #ddd` border, `border-radius: 6px`, `padding: 8px 20px`, `font-size: 14px`, `font-weight: 400`, `background: #fff`, `cursor: pointer`.
Undo button hover: `#007AFF` text, `border-color: #007AFF`.

### FileList Changes — Execution Result Indicator

Add a "状态" column to the file table (between 新文件名 and 扩展名, or as a narrow icon column replacing or alongside the actions column):

| 文件名 | 新文件名 | 状态 | 扩展名 | 大小 | 修改日期 | |
|--------|---------|------|--------|------|---------|---|

Or more minimally: append the result indicator to the new-name column or to the actions (✕) cell. Per D-06, the minimum requirement is per-row visibility.

**Recommended approach (minimal):** Overlay result indicator in the actions cell (last column). Before execution, shows ✕ remove button. After execution, shows ✓ or ✗. If execution results are displayed, the remove button is hidden (user can undo instead).

| State | Indicator | Color | Position |
|-------|-----------|-------|----------|
| Pending (no execution yet) | (remove button ✕) | #999 | Last cell |
| Success | ✓ | #155724 | Last cell, replaces remove button |
| Fail | ✗ | #d32f2f | Last cell, tooltip shows error message |
| After undo | (remove button ✕ restored) | #999 | Last cell, cleared |

Fail tooltip: `<span title="{errorMessage}">✗</span>` — shows the actual error (e.g., "Permission denied", "File not found").

**Visual style:** `font-size: 16px`, `font-weight: 600`, `text-align: center`, `width: 32px`.

## Toast Notification Component

### Behavior
- Position: bottom-right corner, `24px` from screen edges (or right panel bottom-right)
- Z-index: above all panel content (`z-index: 100`)
- Single toast visible at a time (queue new toasts, replace current)
- Auto-dismiss: success 3s, error 5s
- No manual close button (auto-dismiss only — keeps implementation minimal)

### Visual
| Property | Value |
|----------|-------|
| Background | #323232 |
| Text color | #ffffff |
| Font | 14px / 400 |
| Padding | 12px 24px |
| Border-radius | 8px |
| Box-shadow | `0 4px 12px rgba(0,0,0,0.15)` |
| Animation | Slide in from bottom-right (translateY 20px → 0), fade out |

### Message Types

| Type | Icon Prefix | Text | Duration |
|------|-------------|------|----------|
| Success | ✓ (green) | 已成功重命名 {N} 个文件 | 3s |
| Error | ✗ (red) | 重命名失败：{reason} | 5s |
| Partial | ⚠ (yellow) | 已重命名 {N} 个文件，遇到错误已停止。可撤销恢复。 | 5s |
| Undo success | ↩ (neutral) | 已撤销重命名 | 3s |

Icons use inline Unicode with color applied via CSS `span.toast-icon` class.

## Interaction States & Edge Cases

| State | Behavior | Visual |
|-------|----------|--------|
| No files loaded | Right panel shows placeholder (Phase 2 behavior) | Action bar hidden entirely |
| Files loaded, no conflicts | Execute button enabled, action bar visible | Button: accent bg |
| Conflicts exist (D-13) | Execute button disabled, conflict count shown | Button: gray bg, text: #999 |
| User clicks execute | Execute button switches to loading state | "重命名中..." + spinner, disabled |
| Execution succeeds (all files) | Toast: success, FileList rows show ✓, action bar shows undo | ✓ per row, undo button visible |
| Execution fails mid-way (D-07) | Toast: partial error, successful rows show ✓, failed row shows ✗, action bar shows undo | Mixed ✓/✗ per row |
| Execution fails at first file | Toast: error, no rows show ✓, all show ✗ (all failed), action bar shows undo | All ✗ rows |
| User clicks undo | Undo button shows loading briefly (optional), then toast | "已撤销轻命名" toast, rows clear indicators, action bar returns to execute state |
| Undo succeeds | FileList reverts to preview state (new-name column shows preview again), execute button re-enabled | Full revert to pre-execution state |
| File list modified after execution | Clear execution results (files changed — undo may not be valid anymore) | Reset all rows to pending state, hide undo button |
| Rapid double-click execute | Button enters loading state immediately — second click ignored | No visual change on second click |

## Data Flow

```
User clicks "执行重命名"
        ↓
ActionBar calls onExecute callback (prop from App.tsx)
        ↓
(Execution logic in App.tsx:)
  1. Call Tauri fs:rename for each file (two-phase: temp → final)
  2. On success per file: set executionResult[fileId] = "success"
  3. On first error: stop, set executionResult[fileId] = "fail"
  4. Update file list with new names (useFileList.setFiles)
  5. Save undo snapshot to filesystem
        ↓
RenamePanel re-renders:
  - ActionBar shows undo button
  - Toast notification appears
  - FileList per-row ✓/✗ via executionResults prop
        ↓
User clicks "撤销重命名"
        ↓
ActionBar calls onUndo callback
  1. Read undo snapshot from filesystem
  2. Reverse two-phase rename (temp → original)
  3. Restore file list to original names
  4. Clear execution results
  5. Toast: "已撤销重命名"
        ↓
RenamePanel re-renders:
  - ActionBar returns to execute state
  - Row indicators cleared
  - FileList shows preview state again
```

## Interaction Edge Cases

| State | Behavior | Visual |
|-------|----------|--------|
| Execute clicked with empty file list | No-op (execute button should be hidden when no files) | Action bar not rendered |
| Rapid double-click during execution | Second click ignored (button disabled) | No change |
| Execution finishes + user removes a file | Clear execution results — undo may fail due to missing files | Reset to pre-execution state |
| Undo clicked but snapshot file missing (deleted between sessions) | Toast: "无法撤销：未找到撤销快照" | Error toast |
| Files modified externally between phases of two-phase rename | Tauri fs:rename will throw — caught and reported per-file | Per-row ✗ + error in tooltip |
