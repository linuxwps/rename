    创建 `rname/src/hooks/useRenameExecutor.ts`，提供批量重命名执行和撤销的核心逻辑。

    ```typescript
    import { useState, useCallback, useRef } from "react";
    import { rename } from "@tauri-apps/plugin-fs";
    import type { FileItem } from "../types/file";
    import type { PreviewResult, ExecutionResult, ExecutionErrors, FileItemUpdate } from "../types/rename";
    import { saveUndoSnapshot, loadUndoSnapshot, clearUndoSnapshot } from "../undo/snapshot";
    ```

    接口设计：

    ```typescript
    interface UseRenameExecutorReturn {
      isExecuting: boolean;                     // 执行中标志（禁用按钮防重复点击）
      hasExecuted: boolean;                     // 执行已完成标志（显示撤销按钮）
      executionResults: Record<string, ExecutionResult>;  // fileId → 状态
      executionErrors: Record<string, string>;             // fileId → 错误信息
      execute: () => Promise<void>;             // 执行重命名
      undo: () => Promise<void>;                // 撤销重命名
      resetExecution: () => void;               // 重置执行状态（文件列表变更时调用）
    }

    export function useRenameExecutor(
      files: FileItem[],
      previews: PreviewResult[],
      onFileNamesUpdated: (updates: Map<string, FileItemUpdate>) => void
    ): UseRenameExecutorReturn
    ```

    核心逻辑：

    **execute() 方法** — 按 D-08 两阶段策略：

    1. **预验证**（EXEC-02）：
       - 遍历 previews，对每个 `previewResult.newFullName`：
         - 验证不含 `/` 或 `\`（路径穿越防护）
         - 验证 `newFullName` 非空
         - 如果有任何文件名无效，立即拒绝执行并设置错误状态
       - 如果 `previews.length === 0`，直接 return

    2. **两阶段重命名**：
       - 对于每个 file（按 previews 顺序）：
         a. 计算 `dirPath = file.path.slice(0, -file.name.length)` — 提取目录
         b. 计算 `finalPath = dirPath + previewResult.newFullName`
         c. 生成 UUID-based temp 路径: `file.path + ".rntmp." + crypto.randomUUID()`
            - 使用 `crypto.randomUUID()`（Web Crypto API，在 Tauri WebView 中可用）
         d. **Phase 1**: `await rename(file.path, tempPath)` — 原文件 → 临时名
         e. **Phase 2**: `await rename(tempPath, finalPath)` — 临时名 → 最终名
         f. 若成功：`setExecutionResults(prev => ({...prev, [file.id]: "success"}))`
         g. 若任意一步失败（D-07）：
            - 设置当前 file 结果为 "fail"，保存错误信息
            - **停止继续处理后续文件**
            - 已经成功重命名的文件保持现状

    3. **执行后处理**：
       - 如果有至少一个文件成功：
         a. 构造 `Map<string, FileItemUpdate>`，包含所有成功文件的 path/name/extension 更新
            - `newFilePath` = `dirPath + newFullName`
            - `newBaseName` = `newFullName`（不含扩展名）
            - `newExt` = `newExtension`
         b. 调用 `onFileNamesUpdated(updates)` — 更新 App.tsx 中的文件列表
         c. 构造 `UndoSnapshotEntry[]`，保存原始路径 + temp 路径 + 最终路径
         d. 调用 `saveUndoSnapshot(entries)` — 写入磁盘（D-10）
         e. 设置 `hasExecuted = true`
       - 设置 `isExecuting = false`

    **undo() 方法** — 按 D-09/D-12 逆两阶段恢复：

    1. 设置 `isExecuting = true`
    2. 读取快照：
       - `const snapshot = await loadUndoSnapshot()`
       - 如果 null：显示错误（执行器通过控制台错误 + 返回，UI 层处理 toast）
       - 验证快照中的文件路径在当前 file list 中仍然可匹配
    3. **逆两阶段恢复**（注意：按原顺序反向操作）：
       - 对 snapshot.files.reverse() 遍历（逆序恢复避免路径冲突）：
         a. **Phase 1 (reverse)**: `await rename(entry.finalPath, entry.tempPath)` — 最终名 → 临时名
         b. **Phase 2 (reverse)**: `await rename(entry.tempPath, entry.originalPath)` — 临时名 → 原文件名
         c. 如果任意一步失败：记录错误但继续处理下一个文件（最大化恢复）
    4. **撤销后处理**：
       a. 构造 `Map<string, FileItemUpdate>` 恢复原始文件的 path/name/extension
       b. 调用 `onFileNamesUpdated(restoreMap)`
       c. 调用 `clearUndoSnapshot()` — 清除快照（D-11: 单次撤销）
       d. 重置 `hasExecuted = false`
       e. 重置 `executionResults = {}`
       f. 设置 `isExecuting = false`

    **resetExecution() 方法**：
    - 重置 `hasExecuted = false`
    - 重置 `executionResults = {}`
    - 重置 `executionErrors = {}`
    - 不调用 `clearUndoSnapshot()`（快照仍然存在，除非用户触发撤销）

    **状态管理**：
    - `isExecuting` 和 `hasExecuted` 使用 `useState<boolean>`
    - `executionResults` 和 `executionErrors` 使用 `useState<Record<string, ExecutionResult>>` 等
    - 所有状态更新函数使用 `useCallback` 包裹
    - `useRef` 用于存储 `files` 和 `previews` 的最新引用，避免闭包问题
      （execute 函数内的 files/previews 使用 ref 读取当前值）

    **edge cases**:
    - 如果 execute 被重复调用（快速点击）: `isExecuting` 在最顶部检查，如果 true 则直接 return
    - 如果 undo 被重复调用: 类似检查
    - 如果执行完 execute 后用户修改了文件列表（增删文件）: App.tsx 应调用 `resetExecution()`
    - 文件路径中的特殊字符（空格、中文、Unicode）: `rename` API 直接传完整路径即可，Tauri 处理
