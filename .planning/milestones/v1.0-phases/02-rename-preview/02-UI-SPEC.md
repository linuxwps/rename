### Accent Reserved For

Accent (#007AFF) is reserved for these exact elements only:
1. Tab active indicator (bottom border or underline on enabled tab)
2. Enabled tab label text color
3. Form input focus ring (`box-shadow` or `outline`)
4. Checkbox/toggle active state
5. Any clickable label text in enabled-mode forms

Do NOT use accent color for: disabled states, placeholder text, static labels, table borders, background fills, or any decorative element.

**Source:** Pre-populated from existing button color (FileDropZone.css: `#007AFF` picker-button), conflict detection from D-10. Diff colors from D-02 + agent's discretion (RESEARCH.md proposed `#e6ffe6` / `#ffe6e6`; refined to standard Bootstrap alert colors for better readability and WCAG contrast).

### Tab Bar Interaction Design

- **State:** Each tab is a toggle button. State is tracked in `RenameModeStates` (per RESEARCH.md Pattern 4).
- **Visual:** Tab has two visual states:
  - **Enabled (lit):** Accent (#007AFF) bottom border/underline, accent text color, slightly bolder weight
  - **Disabled (unlit):** #666 text color, no bottom border, regular weight
- **Behavior:**
  - Click disabled tab → enable that mode, scroll to/open its form
  - Click enabled tab → disable that mode, hide its form
  - Multiple modes can be enabled simultaneously (D-12)
  - Only one form visible at a time (the last clicked/opened tab)
  - Form input values persist when switching tabs (D-06)
  - When _any_ mode is enabled, a dashed bottom border on the tab bar visually separates it from the forms area
- **Application order:** D-14: 时间→正则→替换→前缀→后缀→扩展名 (left to right)

### Mode State Shape (TypeScript, per RESEARCH.md)

```typescript
interface SequentialConfig {
  enabled: boolean;
  startAt: number;       // default: 1
  digits: number;        // default: 3
  position: "before" | "after";  // default: "after"
}

interface RegexConfig {
  enabled: boolean;
  pattern: string;
  replacement: string;
  caseSensitive: boolean;  // default: false
}

interface PrefixConfig {
  enabled: boolean;
  text: string;
}

interface SuffixConfig {
  enabled: boolean;
  text: string;
}

interface ExtensionConfig {
  enabled: boolean;
  mode: "change" | "remove" | "add";  // default: "change"
  newExtension: string;
}

interface ReplaceConfig {
  enabled: boolean;
  findText: string;
  replaceText: string;
  caseSensitive: boolean;  // default: false
}

interface RenameModeStates {
  sequential: SequentialConfig;
  regex: RegexConfig;
  prefix: PrefixConfig;
  suffix: SuffixConfig;
  extension: ExtensionConfig;
  replace: ReplaceConfig;
}
```

       ↓

       ↓

       ↓

       ↓

       ↓

## Rename Pipeline (Application Order per D-14)

```
Original: "DSC_1234.JPG"

Step 1 (skip if disabled): Sequential  → "DSC_1234_001"
Step 2 (skip if disabled): Regex       → "IMG_1234_001"       (s/DSC_/IMG_/)
Step 3 (skip if disabled): Replace     → "IMG_5678_001"       (1234→5678)
Step 4 (skip if disabled): Prefix      → "holiday_IMG_5678_001"
Step 5 (skip if disabled): Suffix      → "holiday_IMG_5678_001_final"
Step 6 (skip if disabled): Extension   → "holiday_IMG_5678_001_final.png"

Final: "holiday_IMG_5678_001_final.png"
```

Each step is a pure function `(baseName: string, extension: string, config, fileIndex, totalFiles) => { baseName, extension }`. The output of one step feeds into the next.

## Interaction Edge Cases

| State | Behavior | Visual |
|-------|----------|--------|
| No files loaded | Right panel shows "请先在左侧添加文件" placeholder | Centered #999 text |
| Files loaded, no modes enabled | Tab bar visible, no forms open, "新文件名" shows original name | All tabs unlit |
| User types in form | 300ms debounce starts; preview shows stale data during debounce | No loading indicator needed (fast computation) |
| Regex syntax error | Inline error "正则表达式无效" below input; preview treats mode as disabled | Red error text, 13px, below input |
| Conflict detected | Banner shows count; affected rows get red styling | ⚠ icon + red cell border |
| Invalid extension (contains `.`) | Strip the dot automatically | Auto-correct on blur |
| Very long filename (>100 chars) | Truncate with ellipsis in preview column | Same as existing `.file-name` overflow pattern |
| All files removed while forms have input | Forms reset to defaults on next file load? Actually D-06 says preserve input — keep mode state, just hide UI | Right panel returns to placeholder |
